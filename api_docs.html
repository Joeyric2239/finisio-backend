2. Start the server

3. Test the health endpoint

🔄 Job Status Workflow

pending Admin assigns → assigned Cleaner accepts → accepted Cleaner starts → in_progress Cleaner finishes → completed Admin confirms payment → paid
cancelled — from any non-completed status disputed — from in_progress or completed

👤 Users

POST /api/users Create a user (customer / cleaner / admin) ›

Request Body

{
  "name":     "Sarah Mitchell",   // required
  "email":    "sarah@email.sc",   // required, unique
  "role":     "customer",         // required: customer|cleaner|admin
  "phone":    "+248 254 1234",    // optional
  "address":  "Mont Fleuri, Mahé",// optional
  "password": "secret123"         // optional
}

Response 201

{
  "ok": true,
  "data": {
    "id": "uuid...",
    "name": "Sarah Mitchell",
    "email": "sarah@email.sc",
    "role": "customer",
    "verification_status": "unverified",
    "created_at": "2026-04-21T..."
  }
}

If role=cleaner, a cleaner_profiles row is automatically created with approved_status=pending.

GET /api/users/{user_id} Get a user by ID ›

Returns user object. password_hash is never exposed.

Query param: ?role=customer|cleaner|admin when listing GET /api/users

🧹 Cleaners

POST /api/cleaners/{user_id}/approve Admin approves or rejects a cleaner ›

Request Body

{
  "admin_id": "uuid-of-admin",
  "approve": true   // false to reject
}

Side Effects

  • ✓ Sets approved_status = approved/rejected
  • ✓ Sets verification_status on user
  • ✓ Records approved_at timestamp
GET /api/cleaners/{cleaner_id}/jobs Get all jobs assigned to a cleaner ›

Optional filter: ?status=assigned|in_progress|completed

Returns array of booking objects sorted by scheduled_date.

GET /api/cleaners List all approved cleaners. ?approved_only=false to include pending.

📅 Bookings

POST /api/bookings Create a booking → enters pending queue ›

Request Body

{
  "customer_id":    "uuid...",         // required
  "service_type":   "home_deep",       // home_deep | post_construction
  "booking_type":   "subscription",    // subscription | one_time
  "scheduled_date": "2026-05-01",      // required (YYYY-MM-DD)
  "scheduled_time": "09:00",           // optional
  "address":        "Victoria, Mahé", // optional
  "notes":          "Focus on kitchen",// optional
  "media_urls":     ["https://..."],   // optional array
  "hours_booked":   4,                 // optional, deducted from sub
  "subscription_id":"uuid..."          // optional, auto-resolved if omitted
}

Pricing Logic

Subscription booking: SCR 0 charged (covered by monthly plan fee already paid)

One-time home deep: SCR 300/hr

One-time post-construction: SCR 380/hr

Hours are automatically deducted from subscription balance on booking creation.

POST /api/bookings/{id}/assign Admin assigns a cleaner to a pending booking › 
{
  "cleaner_id": "uuid-of-approved-cleaner",
  "admin_id":   "uuid-of-admin"
}

Validates: admin role · booking must be pending · cleaner must be approved. Updates status to assigned.

PATCH /api/bookings/{id}/status Update job status (enforces valid transitions) › 
{
  "status":     "in_progress",   // new target status
  "changed_by": "uuid...",       // user making the change
  "note":       "Started job"    // optional note
}

Valid transitions:

pending → assigned | cancelled
assigned → accepted | cancelled
accepted → in_progress | cancelled
in_progress → completed | disputed
completed → disputed
GET /api/customers/{id}/bookings All bookings for a customer. ?status=pending to filter.
GET /api/bookings/{id}/history Full status change audit log for a booking

💳 Payments

Manual MVP Payment Model: Customers pay via bank transfer or cash. Admin receives proof and manually confirms. System auto-calculates 40/60 commission split on confirmation.
POST /api/payments Record a payment (status=pending) › 
{
  "booking_id":     "uuid...",
  "customer_id":    "uuid...",
  "amount_scr":     1500,
  "payment_method": "bank_transfer",  // bank_transfer | manual | cash
  "reference_no":   "REF-SC-0421"    // optional bank reference
}
POST /api/payments/{id}/confirm Admin confirms payment → auto-calculates commission › 
{
  "admin_id":    "uuid...",
  "reference_no":"REF-SC-0421"  // optional
}

✓ Sets payment status=confirmed

✓ Records confirmed_by and confirmed_at

✓ Auto-creates commission record

✓ Returns {"payment":..., "commission":...}

⭐ Reviews

POST /api/reviews Customer submits review for a completed job › 
{
  "booking_id":  "uuid...",        // must be completed
  "customer_id": "uuid...",        // must be the booking customer
  "rating":      5,                // integer 1–5
  "comment":     "Excellent work!" // optional
}

Automatically recalculates cleaner's average rating after submission.

GET /api/reviews/cleaner/{cleaner_id} All reviews for a cleaner

{
  "sender_id":   "uuid...",

  "message_text":"Hi, I'll arrive at 9am!",
